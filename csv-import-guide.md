# CSV 이슈 가져오기 — 구조 및 사용 가이드

## 개요

`ux_dashboard_4.html` 대시보드에는 CSV 파일로 새 이슈를 추가하거나 기존 이슈를 업데이트할 수 있는 가져오기 기능이 내장되어 있습니다. 별도 서버 없이 브라우저에서 직접 동작합니다.

---

## 업로드 방법

### 1. 버튼 클릭

이슈 테이블 헤더 우측의 **`+ CSV 가져오기`** 버튼을 클릭합니다.

```
[ 전체 | 4 Unusable | 3 Severe | … | 초기화 | + CSV 가져오기 ]
```

### 2. 파일 선택 또는 드래그 앤 드롭

모달이 열리면 두 가지 방법으로 파일을 올릴 수 있습니다.

| 방법 | 설명 |
|---|---|
| **파일 선택** | "파일 선택" 버튼 또는 드롭존 클릭 → 파일 탐색기에서 `.csv` 선택 |
| **드래그 앤 드롭** | `.csv` 파일을 드롭존 위로 드래그해서 놓기 |

### 3. 결과 확인

업로드 완료 후 화면 하단에 토스트 메시지가 표시됩니다.

```
✅ 가져오기 완료 — 신규 3건, 업데이트 1건
```

- **신규**: 기존에 없던 `id`의 이슈가 추가된 건수
- **업데이트**: 기존 `id`와 중복되어 덮어쓴 건수

---

## CSV 파일 형식

### 기본 규칙

- 인코딩: **UTF-8** (Excel에서 저장 시 "CSV UTF-8(쉼표로 분리)" 선택)
- 구분자: `,` (쉼표)
- 첫 번째 행: 반드시 **헤더 행**
- 텍스트 안에 쉼표가 포함될 경우 `"큰따옴표"` 로 감싸기

### 헤더 예시

```csv
번호,분기,퍼널,심각도,UX 문제 유형,문제 상황,이슈 요약,관련 URL,개발 가능 여부,대략적 개발 방향
```

### 데이터 행 예시

```csv
101,2026-2nd,"3. 후보군탐색 (Awareness)","3 Severe","Poor UX Problem","검색 결과 페이지에서 필터 초기화 버튼이 없어 사용자가 조건을 수동으로 하나씩 해제해야 함","필터 초기화 버튼 부재로 재검색 과정이 번거로움","",FALSE,"필터 영역에 초기화 버튼 추가"
```

## 컬럼 매핑 구조

파서는 헤더를 소문자로 변환한 뒤 아래 매핑 테이블에서 내부 필드명을 찾습니다.
**한국어와 영문 헤더를 모두 인식**하며, 인식된 컬럼만 파싱하고 나머지는 무시합니다.

| 내부 필드 | 인식하는 헤더 키워드 | 타입 | 필수 |
|---|---|---|---|
| `id` | `번호`, `id`, `no`, `no.` | 정수 | ✅ |
| `period` | `분기`, `period`, `기간` | 문자열 (예: `2026-1st`) | 권장 |
| `funnel` | `퍼널`, `퍼널 단계`, `퍼널단계`, `funnel` | 문자열 | ✅ |
| `severity` | `심각도`, `severity` | `4 Unusable` / `3 Severe` / `2 Moderate` / `1 Irritant` | ✅ |
| `ux_problem` | `UX 문제 유형`, `ux 문제 유형`, `문제 유형`, `ux_problem` | 문자열 (쉼표로 복수 가능) | ✅ |
| `problem` | `문제 상황`, `문제상황`, `problem` | 문자열 (줄바꿈 포함 가능) | — |
| `problem_rewritten` | `이슈 요약`, `이슈요약`, `problem_rewritten` | 문자열 | ✅ |
| `related_url` | `관련 URL`, `관련url`, `url`, `related_url` | URL 문자열 | — |
| `server_spec` | `개발 가능 여부`, `개발가능여부`, `개발가능`, `server_spec` | `TRUE` / `FALSE` / `예` / `O` / `1` | — |
| `dev_direction` | `대략적 개발 방향`, `개발 방향`, `개발방향`, `dev_direction` | 문자열 | — |

### `period` (분기) 처리 규칙

```
분기 컬럼이 있는 경우  →  해당 분기(KR_DATA 키)에 이슈 배정
분기 컬럼이 없는 경우  →  현재 대시보드에서 선택된 분기에 자동 배정
```

분기 형식은 `YYYY-Nth` 패턴을 권장합니다. 예: `2026-1st`, `2026-2nd`, `2027-1st`
새 분기 키는 자동으로 생성되며 상단 드롭다운에도 즉시 반영됩니다.

---

## 중복 처리 방식

같은 분기 내에 동일한 `id`가 이미 존재하면 **기존 데이터를 CSV 값으로 덮어씁니다.**
`images` 필드는 CSV에 포함되지 않으므로 덮어쓰기 시에도 기존 이미지는 유지됩니다.

```
동일 period + 동일 id  →  기존 항목 업데이트 (이미지 유지)
동일 period + 새 id    →  신규 항목으로 추가
새 period              →  새 분기 키 생성 후 추가
```

## 데이터 흐름 요약

```
CSV 파일 선택 or 드롭
        ↓
parseCSVText()
  - 헤더 파싱 (소문자 변환 후 CSV_COL_MAP 조회)
  - 각 행을 내부 issue 객체로 변환
  - server_spec: "TRUE"/"예"/"O"/"1" → true, 나머지 → false
        ↓
importCsvData()
  - issue.period 기준으로 KR_DATA[period] 배열에 병합
  - 중복 id → 덮어쓰기 / 신규 id → push
  - buildPeriodDropdown() 및 selectPeriod() 호출로 UI 즉시 갱신
        ↓
대시보드 전체 재렌더링
  - KPI 수치, Funnel 차트, 문제 유형 차트, 이슈 테이블 모두 반영
```

## CSV 양식 다운로드

모달 내 **"양식 다운로드"** 버튼을 누르면 올바른 헤더와 예시 1행이 포함된 `UX_이슈_양식.csv` 파일이 다운로드됩니다. Excel 또는 Google Sheets에서 열어 작성 후 저장하면 됩니다.

> Excel 저장 시 반드시 **"CSV UTF-8(쉼표로 분리)(*.csv)"** 형식으로 저장하세요.
> 일반 "CSV(쉼표로 분리)"는 한글이 깨질 수 있습니다.

---

## 주의 사항

- 업로드한 데이터는 **브라우저 메모리에만 저장**됩니다. 페이지를 새로고침하면 초기 하드코딩 데이터로 돌아옵니다.
- 이슈 이미지(`images`)는 CSV 형식의 특성상 포함되지 않습니다. 이미지는 기존 HTML 내 데이터를 그대로 유지합니다.
- 한 번에 여러 분기의 이슈를 담은 CSV도 가능합니다. `분기` 컬럼 값이 다르면 각각의 분기에 자동으로 분류됩니다.
