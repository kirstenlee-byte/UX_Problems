# UX Problems 프로젝트 가이드

## 엑셀 파일 업데이트 시 작업 패턴

엑셀 파일(`[KR & JP] UX 과락 총정리 _ UX欠陥リストアップ (3).xlsx`)이 업데이트되면 아래 절차를 따른다.

### 1. 엑셀 데이터 분석
- KR 탭, JP 탭 각각의 과락 이슈 목록을 확인한다.
- 각 이슈의 **origin 태그** (1st party / 3rd party / unknown 등)를 파악한다.
- 이슈별 **증빙 이미지** 삽입 여부를 확인한다.

### 2. ux_dashboard_4.html 업데이트
- **상단 차트**: KR·JP 탭 각각의 과락 이슈 건수를 올바르게 반영한다.
- **KR 탭 / JP 탭 이슈 수 일치**: 탭 간 이슈 수가 불일치하지 않도록 한다.
- **origin 태그**: 각 이슈에 올바른 origin (1st party / 3rd party 등)을 태깅한다.
- **이미지**: 증빙 스크린샷이 있는 이슈는 이미지를 삽입한다.

### 3. 브랜치 및 PR
- 브랜치명: `claude/[작업내용]-[session-id]`
- 변경사항은 커밋 후 push → PR 생성

### 주요 이슈 유형 (과락 기준)
- UX 과락 = 사용성 기준 미달 이슈
- origin 분류: `1st party` (자사 문제) / `3rd party` (외부 문제) / 태그 없음 주의
